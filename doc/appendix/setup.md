# 부록 A 개발 도구 설치

이 책의 실습은 몇 가지 개발 도구가 컴퓨터에 설치돼 있다는 전제 위에서 진행된다.
이 부록에서는 Git, Go, sqlite3를 macOS·윈도우·리눅스별로 설치하는 방법을 정리하고, 선택 도구인 VS Code와 Claude Code의 설치도 안내하겠다.
이미 도구를 갖춘 독자라면 마지막의 "한 번에 확인하기" 절에서 버전만 확인하고 지나가도 된다.
설치 절차와 명령은 도구가 개정되며 달라질 수 있으므로, 여기 적힌 방법이 막히면 각 공식 사이트의 안내를 우선하자.

모든 설치와 확인 명령은 터미널에 입력한다.

::: info [용어 풀이] 터미널(Terminal)
마우스 대신 키보드로 명령을 입력해 컴퓨터를 조작하는 문자 기반 창이다.
macOS에는 "터미널", 윈도우에는 PowerShell이라는 이름으로 기본 설치돼 있고, 리눅스 데스크톱에도 같은 이름의 앱이 있다.
이 책에 나오는 `go run ./cmd/server` 같은 명령은 모두 이 창에 입력하고 엔터를 쳐서 실행한다.
:::

여는 법은 이렇다.
macOS는 ⌘+스페이스를 눌러 "터미널"을 검색해 실행하고, 윈도우는 시작 메뉴에서 "PowerShell"을 검색해 실행하며, 우분투 리눅스는 Ctrl+Alt+T를 누른다.
창이 뜨면 커서가 깜빡이는 자리에 명령을 한 줄 입력하고 엔터를 치면 된다.

윈도우 독자에게는 한 가지 권하고 싶은 것이 있다.
이 책 2부에는 macOS·리눅스의 셸을 전제한 명령이 몇 개 있으므로(18장의 direnv, 20장의 백업 명령 등), WSL(Windows Subsystem for Linux, 윈도우 안에서 우분투 리눅스를 돌려 주는 윈도우의 공식 기능)을 켜고 그 안에서 실습하면 책의 명령을 그대로 따라 할 수 있다.
WSL 없이 PowerShell만으로도 1부는 문제없이 진행되니, 지금 당장 결정하지 않아도 된다.

## 무엇이 언제 필요한가

책 전체에서 각 도구가 처음 쓰이는 자리를 먼저 짚어 두겠다.
전부를 미리 설치할 필요는 없고, 해당 장에 도착했을 때 이 부록으로 돌아와 필요한 것만 설치해도 된다.

| 도구 | 처음 쓰는 곳 | 용도 |
|---|---|---|
| sqlite3 | 4장 | SQL 실습 |
| Git | 13장 | 저장소를 내 컴퓨터로 복사(클론) |
| Go | 13장 | 앱 서버 실행 |
| Claude Code(선택) | 2장 | AI 코딩 에이전트 |
| VS Code(선택) | 13장 | 코드 편집기 |

2부에서 필요해지는 GitHub·Vercel·Supabase는 설치할 프로그램이 아니라 가입할 서비스이므로, 각 장에서 그때그때 안내한다.

## 설치 방법을 고르는 기준

설치 방법은 크게 두 갈래다.
공식 사이트에서 설치 파일을 내려받아 실행하는 방법과, 패키지 매니저로 명령 한 줄에 설치하는 방법이다.

::: info [용어 풀이] 패키지 매니저(Package Manager)
프로그램의 내려받기, 설치, 업그레이드를 명령 한 줄로 처리해 주는 도구다.
휴대폰의 앱 스토어를 명령줄로 옮겨 놓은 셈이다.
macOS의 Homebrew, 윈도우의 winget, 우분투 리눅스의 apt가 대표적이다.
:::

공식 설치 파일은 어느 컴퓨터에서나 절차가 같고 항상 최신 버전을 준다.
패키지 매니저는 설치가 명령 한 줄로 끝나고 나중에 여러 도구를 한꺼번에 업그레이드할 수 있어 편하지만, 저장소에 따라 등록된 버전이 공식 최신보다 늦을 수 있다.
특히 우분투 기본 저장소의 Go는 몇 단계 낡은 버전인 경우가 많아 아래에서 따로 짚는다.
윈도우의 winget은 윈도우 11과 최신 윈도우 10에 기본 내장돼 있고, macOS의 Homebrew는 기본 내장이 아니므로 쓰려면 brew.sh 의 안내를 따라 먼저 설치해야 한다.
둘 중 무엇을 골라도 결과는 같으니, 아래에서는 두 방법을 함께 적어 두겠다.

## Git

Git은 코드의 변경 이력을 관리하는 버전 관리 도구다.
개념은 14장에서 다루지만, 저장소를 내 컴퓨터로 복사하는 클론(clone)에 쓰이므로 13장의 실습 전에 설치가 필요하다.

- **macOS**: 터미널에서 `git`을 처음 실행하면 명령줄 개발자 도구를 설치하겠느냐는 창이 뜨는데, 설치를 누르면 Git이 함께 들어온다. Homebrew를 쓴다면 `brew install git`으로도 된다.
- **윈도우**: `winget install --id Git.Git`을 실행하거나, git-scm.com에서 Git for Windows 설치 파일을 내려받아 실행한다.
- **우분투 리눅스**: `sudo apt install git`을 실행한다.

명령 앞의 `sudo`는 관리자 권한으로 실행하라는 뜻이다.
컴퓨터 로그인 비밀번호를 물어보는데, 보안상 입력해도 화면에는 아무 글자도 나타나지 않으니 당황하지 말고 그대로 치고 엔터를 누르면 된다.

설치가 끝났으면 커밋 이력에 남을 이름과 이메일을 등록해 둔다.
다음 두 명령은 그 값을 전역 설정에 저장하는 것으로, 14장에서 커밋을 만들 때 필요하다.

```bash
git config --global user.name "이름"
git config --global user.email "이메일 주소"
```

GitHub 계정을 만들 예정이라면 계정에 쓸 이메일과 같은 주소를 적어 두는 편이 좋다.

## Go

Go는 이 앱을 만드는 언어다.
공식 다운로드 페이지(go.dev/dl)가 macOS·윈도우·리눅스용 설치 파일을 모두 제공하므로, 어느 OS든 이 방법이 가장 확실하다.

- **macOS**: go.dev/dl 의 설치 파일을 실행하거나, `brew install go`를 실행한다.
- **윈도우**: `winget install --id GoLang.Go`를 실행하거나, go.dev/dl 의 설치 파일을 실행한다.
- **우분투 리눅스**: 기본 저장소의 버전이 낡은 경우가 많으므로 go.dev/dl 에서 압축 파일을 받아 직접 푸는 방법을 권한다. 설치 파일을 실행하는 macOS·윈도우와 달리 두 단계가 필요하다.

리눅스에서 압축 파일로 설치할 때는 먼저 받은 파일을 `/usr/local`에 푼다(파일 이름의 버전은 내려받은 것에 맞춰 바꾼다).

```bash
sudo tar -C /usr/local -xzf go1.26.0.linux-amd64.tar.gz
```

여기까지만 하면 `go` 명령이 아직 동작하지 않는다.
푼 폴더를 터미널이 명령을 찾아다니는 목록에 등록해야 한다.

```bash
echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.bashrc
```

이 한 줄을 넣고 터미널을 닫았다 다시 열면 `go` 명령이 어디서든 실행된다.

::: info [용어 풀이] PATH와 셸 설정 파일
PATH는 터미널이 명령을 찾아다니는 폴더 목록이다.
`go`라고만 쳐도 실행되는 것은 Go가 설치된 폴더가 이 목록에 들어 있기 때문이고, 목록에 없으면 프로그램이 설치돼 있어도 "명령을 찾을 수 없다"는 오류가 난다.
목록에 폴더를 더하려면 터미널을 열 때마다 자동으로 실행되는 셸 설정 파일에 위와 같은 한 줄을 추가하고 터미널을 다시 연다.
그 파일은 홈 폴더의 `~/.bashrc`(bash) 또는 `~/.zshrc`(zsh)이고, 지금 쓰는 셸이 무엇인지는 `echo $SHELL`로 확인할 수 있다(zsh를 쓴다면 위 명령의 `~/.bashrc`를 `~/.zshrc`로 바꾼다).
:::

패키지 매니저로 설치한 버전이 조금 낡아도 크게 걱정할 일은 아니다.
Go 1.21부터는 프로젝트의 `go.mod`가 더 새로운 버전을 요구하면, 설치된 Go가 해당 툴체인을 자동으로 내려받아 사용한다.

설치를 마쳤으면 버전을 확인한다.

```bash
go version
```

## sqlite3

sqlite3는 SQLite 데이터베이스 파일을 열어 SQL을 실행하는 명령줄 도구로, 4장의 SQL 실습과 13장에서 로컬 데이터베이스 파일을 들여다볼 때 쓴다.

- **macOS**: 기본으로 들어 있어 설치할 것이 없다.
- **윈도우**: `winget install --id SQLite.SQLite`를 실행하거나, sqlite.org의 다운로드 페이지에서 sqlite-tools 압축 파일을 받아 푼다. 압축을 푼 경우에는 그 폴더에서 터미널을 열거나, 앞의 Go 절에서 본 PATH에 그 폴더를 추가해야 `sqlite3` 명령이 어디서든 실행된다.
- **우분투 리눅스**: `sudo apt install sqlite3`를 실행한다.

설치를 마쳤으면 버전을 확인한다.

```bash
sqlite3 --version
```

## 선택 도구: VS Code와 Claude Code

다음 두 도구는 없어도 책을 따라갈 수 있지만, 있으면 2·3장과 13장의 이야기를 직접 체험할 수 있다.

VS Code는 code.visualstudio.com 에서 OS에 맞는 설치 파일을 내려받아 설치한다.
이 저장소에는 권장 확장 목록(`.vscode/extensions.json`)이 들어 있어, 저장소를 열면 VS Code가 필요한 확장의 설치를 알아서 제안한다.
이 구성은 13장에서 해부한다.

Claude Code는 2장에서 다루는 AI 코딩 에이전트다.
공식 가이드가 권장하는 방법은 OS별 설치 스크립트 실행이다.

macOS·리눅스(윈도우 안에서 리눅스를 돌리는 WSL 포함)는 터미널에서 다음 명령을 실행한다.
설치 스크립트를 내려받아 바로 실행하는 명령이다.

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

윈도우는 PowerShell에서 다음 명령을 실행한다.

```powershell
irm https://claude.ai/install.ps1 | iex
```

패키지 매니저를 쓴다면 macOS는 `brew install --cask claude-code`, 윈도우는 `winget install Anthropic.ClaudeCode`로도 설치할 수 있다.
다만 설치 스크립트로 설치하면 이후 새 버전이 자동으로 반영되는 반면, 패키지 매니저로 설치하면 업그레이드 명령을 직접 실행해야 한다.

설치 후 터미널에서 `claude`를 실행하면 로그인 안내가 나온다.
이용에는 Anthropic 계정이 필요하고, 유료 구독이나 API 사용량 과금 중 한 가지 방식으로 요금이 발생한다는 점은 미리 알아 두자.

## 한 번에 확인하기

설치를 마쳤다면, 다음 명령들이 각각 버전을 출력하는지 확인한다.

```bash
git --version
go version
sqlite3 --version
```

성공하면 이런 모양의 줄이 세 번 나온다.

```
git version 2.43.0
go version go1.26.0 darwin/arm64
3.45.1 2024-01-15 ...
```

버전 번호와 뒤에 붙는 정보는 컴퓨터마다 다르니, 숫자가 위와 달라도 걱정할 것 없다.
세 명령 모두 이렇게 버전을 출력하면 이 책의 모든 실습을 진행할 준비가 끝난 것이다.

명령을 찾을 수 없다는 오류가 나오면 원인은 셋 중 하나다.
설치가 아직 안 됐거나, 설치 직후라 터미널을 새로 열어야 반영되는 경우이거나, PATH에 설치 폴더가 등록되지 않은 경우다.
먼저 창을 닫고 다시 열어 보고, 그래도 같은 오류가 나면 앞의 "PATH와 셸 설정 파일" 상자로 돌아가 설치한 폴더가 PATH에 들어 있는지 확인하자.
참고로 이 책의 코드는 Go 1.26에서 확인했다.

## 정리

이 부록에서는 책의 실습에 필요한 도구를 설치했다.

필수 도구는 셋이다.
SQL 실습에 쓰는 sqlite3, 저장소를 복사해 오는 Git, 앱 서버를 돌리는 Go다.
설치 방법은 공식 설치 파일과 패키지 매니저 두 갈래가 있고, 우분투 기본 저장소의 Go처럼 패키지 매니저의 버전이 낡은 경우만 주의하면 어느 쪽을 골라도 좋다.
선택 도구인 VS Code와 Claude Code까지 갖추면 2·3장과 13장의 개발 환경 이야기를 그대로 재현할 수 있다.

도구가 준비됐으니, 4장의 SQL 실습이든 13장의 앱 실행이든 본문 어디서든 명령이 나오면 바로 따라 할 수 있다.
